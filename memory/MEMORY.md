# 메이드아트 프로젝트 메모

최종 갱신: 2026-06-02 05:02 KST

## 프로젝트 정체성

`D:\Claude_works\made_art`는 오둥이 캐릭터 기반의 프롬프트 빌더 프로젝트다. 목표는 사용자가 선택한 항목을 바탕으로 이미지 생성, 음악 생성, 4컷카툰 생성에 바로 붙여넣을 수 있는 영어 프롬프트 txt를 만드는 것이다.

## 핵심 구성

- `index.html`: React 18 UMD + Babel standalone으로 JSX 파일을 직접 로드하는 단일 페이지 앱 진입점.
- `app.jsx`: 앱 쉘, 상단 탭, 테마/tweaks, 공통 컴포넌트.
- `image-tab.jsx`: 그림 생성 프롬프트 탭. 현재는 주제, 묘사, 빛, 카메라, 심도, 비율 중심으로 간소 구현되어 있다.
- `music-tab.jsx`: 명상과 치유 중심 음악 생성 프롬프트 탭. 핸드팬, 추천 악기, 자연음, 템포, 스케일, 특수 지시문을 프롬프트에 반영한다.
- `data.js`: 캐릭터 프리셋, 시간대, 날씨, 배경, 카메라, 음악, 탈무드 카테고리/스토리 데이터.
- `odungi-agents.js`: CLI 자동 생성기. `node odungi-agents.js 이미지|음악|4컷`으로 `산출물` 폴더에 txt 프롬프트를 생성한다.
- `styles.css`: 전체 UI 스타일. 4컷카툰 관련 스타일 흔적도 포함되어 있다.
- `메이드아트_빌드.txt`: 요구사항과 작업 이력이 담긴 사실상 프로젝트 빌드 노트.
- `AI_INSTRUCTIONS.md`: 코드 수정 후 자동 git commit/push 지시가 들어 있다.

## 현재 상태

- 웹 UI 상단 탭은 `그림`, `음악` 2개만 연결되어 있다.
- 4컷카툰은 `data.js`, `styles.css`, `odungi-agents.js`에는 데이터/로직 흔적이 있지만 웹 UI 탭으로는 아직 연결되어 있지 않다.
- 그림 탭은 원래 요구사항보다 축소된 상태다. 캐릭터, 시간대, 날씨, 배경, 화풍, 기법, 포커싱, 아웃포커싱 선택 UI가 아직 본격 반영되지 않았다.
- 음악 탭은 `명상과 치유` 방향으로 많이 구현되어 있다.
- `odungi-agents.js`는 `node --check` 문법 검사를 통과했다.
- `package.json`에는 의존성만 있고 실행 스크립트는 없다. 현재 앱은 `index.html`을 브라우저로 여는 방식이다.

## 중요한 요구사항

- 영어 프롬프트에 한국어 캐릭터 이름을 그대로 넣지 않도록 주의한다. 이미지 모델이 야채/김치를 그릴 수 있으므로 종/외형 설명 중심으로 작성한다.
- 동물 캐릭터 5명(배추, 상추, 열무, 까미, 김치)은 의인화해서 두 발로 서는 장면 연출이 가능하다.
- 이미지 생성의 포커싱/아웃포커싱 항목에는 출연 캐릭터뿐 아니라 배경도 선택 대상에 포함되어야 한다.
- 4컷카툰은 9:16 세로 구성, 4컷 한 장, 흰색 테두리, 랜덤 1컷 분할 클로즈뷰, 중복 없는 다양한 카메라뷰가 핵심이다.
- 음악 프롬프트 첫 줄에는 첨부 이미지의 감정과 느낌을 음악에 반영하라는 지시가 들어간다.

## git/파일 주의사항

- 현재 확인된 git 상태 기준으로 작업트리는 깨끗하지 않았다.
- 추적 안 된 항목으로 `.bkit/`, `node_modules/`, `package.json`, `package-lock.json`, `NUL` 등이 있었다.
- `NUL`은 Windows 예약 이름이라 git, 압축, 복사 작업에서 문제를 만들 수 있으므로 정리 후보다.
- `.gitignore`가 보이지 않았다. `node_modules/`, `.bkit/`, `산출물/`, `NUL` 제외 규칙을 검토하는 것이 좋다.
- 민감정보는 메모나 git에 남기지 않는다.
- GitHub 계정은 `elsa9598`을 사용한다.
- GitHub 푸시 토큰은 `D:\.env`의 `GITHUB_API_KEY`를 그때그때 읽어 사용한다. 토큰 값은 출력, 기록, 커밋하지 않는다.
- 이 프로젝트는 작업 완료 후 항상 필요한 파일만 선별해 커밋하고 푸시한다.
